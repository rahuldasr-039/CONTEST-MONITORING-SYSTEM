⚙️ How It Works (The Faculty Workflow)Assigning Work: An HOD creates an assignment matching an upcoming global contest (e.g., "LeetCode Weekly").Student Attempt: Students participate in the contest on the official platform.Live Verification: When the deadline passes, the HOD or Mentor opens the "Contest Monitor" tab. The backend rapidly triggers scrapers.py, checks every pending student's API history using the Smart Matcher, and updates their status to "Completed" if they participated.Actionable Data: The Mentor clicks "Download Defaulters" to get a clean CSV of everyone who missed the contest, ready for email follow-ups.⚠️ TroubleshootingOperationalError (no such column): This happens if you modify the database models in app.py but don't reset the physical database. Simply stop the server and run python setup.py to rebuild it.Scraping Timeouts: CodeChef actively blocks automated scripts. The scrapers.py file includes a custom User-Agent header to bypass basic 403 Forbidden errors, but excessive rapid refreshing may still trigger rate limits.
